\section{Data Inputs}
The main data input to the propagation of satellites in this project is Two-Line Element sets (TLEs). TLEs are a common publicly accessible format which describes Earth-orbiting satellite positions. The primary source of TLE data is SpaceTrack \cite{spacetrack}. Although SpaceTrack is the primary source, many projects such as this project and other research rely on sources such as CelesTrak, which takes the data from SpaceTrack and organizes it into specific groupings \cite{celestrakTLE}. CelesTrak updates the majority of its TLE files approximately every two hours. Its Starlink data can expect multiple updates per day, making it suitable for this project's application \cite{usradioguyNewTLE}.

\section{Two Line Element Data}
TLEs are a standardized format used to describe the orbital properties of Earth-orbiting satellites. A TLE represents a satellite's orbit at a specific moment in time, known as the epoch. Because the TLE format is simple and lightweight, it is widely supported and remains the most common method for distributing orbit data for tracking and prediction.

A TLE consists of three lines: line 0 (optional) contains the satellite name, line 1 contains identification and basic orbital parameters, and line 2 describes remaining orbital elements such as inclination. This is summarized in Table~\ref{tab:summaryOfTle}. An example of TLE data is shown in Listing~\ref{fig:tle-starlink-5694}.

\section{Orbit Propagation}
The Simplified General Perturbations Model 4 (SGP4) \cite{vtSpacetrk} is used for orbit propagation.

\section{Coordinate Transformations} \label{section:coordinateTransformations}
Coverage estimation first requires the satellite and observer to be expressed in a common coordinate system at a given epoch. Satellites propagated using SGP4 are expressed in the True Equator Mean Equinox (TEME) frame, while ground sites are typically given in geodetic coordinates. It is therefore necessary to convert both to an Earth-Centered Earth-Fixed (ECEF) frame \cite{asteRisk2023, astropy:2022}.

\subsection{TEME}
TEME is the native output frame of SGP4. It is non-rotating with respect to Earth's surface and must be converted before comparison with geodetic locations \cite{asteRisk2023}.

\subsection{ECEF}
ECEF is a rotating Cartesian frame whose origin is at Earth's center of mass. The axes align with Earth's rotation and geometry \cite{Navipedia_CTRS_2011}. This is shown in Figure~\ref{fig:ECEF}.

\section{Satellite Visibility Model}
Visibility is evaluated using line-of-sight (LOS) analysis. The vector from observer to satellite is

\begin{equation}
d = r_{sat} - r_{obs}.
\end{equation}

The LOS vector is transformed into the East-North-Up (ENU) frame, enabling calculation of the elevation angle \cite{Navipedia_ECEF_ENU_2011}.

\begin{equation}
\theta_{el} = \arctan \left( \frac{u}{\sqrt{e^2 + n^2}} \right)
\end{equation}

Only satellites exceeding a $15^\circ$ elevation mask are considered visible \cite{Washburn_EarthCov_2004}.

\section{Beam-width Model}
Visibility alone does not guarantee service. A simplified nadir-pointing beam model is used, where a user must lie within a maximum off-nadir angle $\theta_{beam}$. Only satellites satisfying both elevation and beam-width constraints are deemed capable of providing coverage.

\section{Coverage Window Detection}
Coverage windows are determined by evaluating satellite visibility at discrete time steps. A window begins when coverage becomes available and ends when it is lost. For each window, the system records start time, end time, duration, and optimal satellite selection.
